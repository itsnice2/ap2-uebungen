REST - Representational State Transfer

Frage an ChatGPT: Erkläre mir die Rest Architektur in der Software Entwicklung.

REST ist ein Architekturstil, kein Protokoll. Nicht jede REST-API erfüllt alle 6 Anforderungen vollständig.

  1. Client-Server-Architektur
    2.
    • Client (z.B. Web-App) und Server (Backend) sind getrennt.
    • Der Client stellt Anfragen, der Server gibt Antworten zurück.
  2. Statelessness - Zustandslosigkeit
    3.
    • Jede Anfrage muss alle notwendigen Informationen enthalten.
    • Der Server speichert keine Informationen über frühere Anfragen (keine Sitzung).
  3. Cacheability
    4.
    • Antworten sollten angeben, ob sie vom Client zwischengespeichert werden dürfen.
    • Das erhöht die Leistung und reduziert unnötige Anfragen.
  4. Uniform Interface - Einheitliche Schnittstelle
    5.
    • Einheitliche und konsistente Kommunikation mit Ressourcen.
    • Dazu gehören:
      • Ressourcen mit URIs (z.B. /users/123)
      • Standard-HTTP-Methoden wie GET, POST, PUT, DELETE
      • Repräsentationen von Daten (z.B. JSON, XML)
      • Hypermedia (HATEOAS) (optional): Links in Antworten zur Navigation
  5. Layered System - Schichtenarchitektur
    6.
    • Das System kann aus mehreren Schichten bestehen (z.B. Load Balancer, Proxies).
    • Der Client erkennt nicht, ob er direkt mit dem Endserver kommuniziert.
  6. Code on Demand (optional)
    7.
    • Der Server kann Code (z.B. JavaScript) an den Client senden, der dort ausgeführt wird.
