Geläufige Datenbanktypen

Übersicht über ganz viele DB-Engines

Typ Erklärung Beispiele
Relationale Datenbanken Eine relationale Datenbank (RDB) ist eine Möglichkeit, Informationen strukturiert in Tabellen, Zeilen und Spalten zu speichern. Eine RDB bietet die Möglichkeit, durch das Zusammenführen von Tabellen Verknüpfungen zwischen Informationen herzustellen, sodass die Beziehungen zwischen verschiedenen Datenpunkten erkennbar werden und daraus Erkenntnisse gewonnen werden können. MariaDB, MySQL, PostgreSQL, Microsoft SQL Server, Oracle Database
Objektorientierte Datenbanken

Eine Datenbank, die auf dem Objektdatenbankmodell basiert. Im Unterschied zur relationalen Datenbank werden Daten hier als Objekte im Sinne der Objektorientierung verwaltet.

Ein Objekt modelliert normalerweise einen Gegenstand oder Begriff und enthält insbesondere dazugehörige Attribute; so gehört zum Beispiel die Farbe und das Gewicht eines Autos zu dem Objekt Auto. Attribute beschreiben ein Objekt näher. Daten und Methoden (die Funktionen zum Zugriff auf die Daten) werden in den Objekten zusammen abgelegt.

 InterSystems IRIS, GridGain, Actian NoSQL Database, ObjectStore, Db4o
Dokumentenorientierte Datenbanken Eine Datenbank, bei der Dokumente die Grundeinheit zur Speicherung der Daten bilden. Couchbase, MongoDB. BaseX, Elasticsearch, eXist, MarkLogic Server, RavenDB, SimpleDB, NoSQL
Zeitreihen-Datenbanken Eine Datenbank, die für das Speichern und die Analyse von Zeitreihen wie Sensordaten oder Aktienkursen optimiert ist. Anwendungsdomänen sind zum Beispiel das Internet der Dinge, der Kapitalmarkt, das Monitoring von IT-Systemen oder die Forschung auf physiologischen Signalen. InfluxDB, Prometheus, Graphite
