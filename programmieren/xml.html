XML - Extensible Markup Language

Eine Auszeichnungssprache, mit der man Daten strukturieren, speichern und übertragen kann, ähnlich wie HTML, die aber nicht zur Darstellung von Webseiten gedacht ist, sondern eher zur Datenbeschreibung.

Mit eigenen definierten Tags ist es eine leistungsstarke Methode, um Daten in einem Format zu haben, das gespeichert, durchsucht und geteilt werden kann.

Beispiel

Attribute

Elemente können auch Attribute enthalten, die zusätzliche Informationen über das Element bereitstellen. Attribute sind in einem Starttag enthalten und haben einen Namen und einen Wert.
<!- ... -> Kommentar (optional)

Verarbeitungsanweisungen / Processing Instructions (optional)

Anweisungen, die spezielle Informationen für die Verarbeitung des Dokuments enthalten können

Regeln

für Tags

  • XML-Tag-Namen stehen in spitzen Klammern.
  • XML-Tag-Namen beachten die Groß- und Kleinschreibung (case sensitive).
  • Elemente in einem XML-kompatiblen Dokument werden von öffnenden und schließenden Tags umschlossen.
  • Die Tags müssen korrekt verschachtelt sein.
  • Alle XML-Dokumente müssen ein Wurzelelement (root) haben.
  • Alle anderen Elemente müssen sich innerhalb dieses Wurzelelements befinden.

für Elemente

  • Namen können Buchstaben, Zahlen und andere Zeichen beinhalten.
  • Namen dürfen nicht mit einer Nummer oder einem Interpunktionszeichen beginnen.
  • Namen dürfen nicht mit der Buchstabenkombination xml oder XML beginnen.
  • Namen dürfen keine Leerzeichen enthalten.
